# API文档规范

## 文档格式

使用 OpenAPI 3.0 规范，采用 YAML 格式编写API描述文件。

## 文件存放位置

API描述文件统一存放在后端项目根目录的 `openapi/` 目录下。

核心API文档放在 `openapi/spec/` 目录，插件扩展的API文档放在插件自身目录下。

## 文档内容

每个API包含：接口路径、请求方法、参数说明、请求体格式、响应格式、错误状态码说明。

路径按照资源分组，使用统一的前缀版本号。

## 工具集成

开发环境启动服务时，自动提供OpenAPI UI界面，方便在线查看和测试API。

## 版本管理

API路径包含版本前缀，如 `/api/v1/`，方便后续API版本升级。

## 生成方式

OpenAPI文档手动编写维护，保证文档质量。Prisma模型可以参考，但不自动生成文档。
