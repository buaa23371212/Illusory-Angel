# 后端技术选型

## 开发语言

选择 TypeScript 作为开发语言，前后端使用同一种语言，开发效率更高，减少上下文切换成本。

## 运行环境

选择 Node.js 作为后端运行环境，生态丰富，入门门槛低，适合个人开发。

## 开发框架

选择 Express 框架，轻量灵活，生态成熟，文档完善，学习成本低，适合中小型项目开发。

## 数据库

选择 MySQL 作为主数据库，关系型数据模型适合本项目的数据结构，社区支持好，稳定性高。

## ORM 选型

使用 Prisma 作为 ORM 工具，类型安全，自动生成代码，开发效率高，支持 MySQL，使用体验好。

## 接口规范

采用 RESTful API 设计风格，路由命名清晰，易于理解和维护，前端对接简单。

## 插件机制

插件通过目录扫描自动加载，每个插件独立目录存放，包含自身的模型、路由和业务逻辑，不修改核心代码即可扩展功能。

## 依赖管理

使用 npm 进行依赖管理，遵循语义化版本规范，定期更新依赖包。
