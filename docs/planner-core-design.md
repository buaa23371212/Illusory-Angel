# planner core - 设计文档

## 项目概述

planner是一个帮助用户规划和管理目标的工具。用户可以创建项目，在每个项目中定义多个目标，并跟踪这些目标的进度。

## 核心数据结构设计

### 整体架构

项目采用简洁层级数据结构：
- **项目（Project）** - 最高层级组织单元，包含多个目标，也可以有约束（全局约束）
- **目标（Goal）** - 具体的目标，隶属于某个项目
- **约束（Constraint）** - 附加在项目或目标上的条件限制

### 项目（Project）

#### 核心属性

- **ID**：唯一标识符，用于索引和关联
- **名称**：项目名称，用户自定义
- **描述**：项目简介说明
- **种类**：项目类型
- **创建时间**：记录项目创建时间
- **更新时间**：记录最后修改时间
- **状态**：活跃/已完成/已存档
- **排序权重**：支持用户自定义排序

#### 关系

一个项目可以包含 **零个或多个** 目标。  
一个项目可以有 **零个或多个** 约束（全局约束）。

### 目标（Goal）

#### 核心属性

- **ID**：唯一标识符
- **所属项目ID**：关联到父项目
- **名称**：目标名称
- **描述**
- **状态**：未开始/进行中/已完成/已暂停
- **创建时间**：目标创建时间
- **更新时间**：最后修改时间

#### 关系

- 一个目标必须 **属于且仅属于** 一个项目
- 一个目标可以有 **零个或多个** 约束

### 约束（Constraint）

约束是附加在目标上的条件限制。

#### 核心属性

- **ID**：唯一标识符
- **所属目标ID**：关联到目标
- **参数**：约束参数（JSON格式存储，根据类型不同而不同）
- **描述**：约束说明

#### 关系

一个约束必须 **属于且仅属于** 一个所有者（项目或目标）。

## 数据操作接口设计

### 项目操作

- 创建项目
- 读取项目列表（支持分页、按名称搜索）
- 读取单个项目详情
- 更新项目信息
- 删除项目（级联删除下属所有目标和约束）
- 添加约束到项目
- 获取项目所有约束

### 目标操作

- 创建目标（必须关联项目）
- 读取目标列表（可按项目筛选，可按优先级排序）
- 读取单个目标详情（包含所有材料需求和特殊条件）
- 更新目标信息
- 更改目标状态
- 删除目标（级联删除下属所有约束）
- 批量移动目标到其他项目

### 约束操作

- 添加约束到所有者（项目或目标）
- 更新约束参数
- 删除约束
- 获取所有者的所有约束
