## 约束数据结构

本插件定义5种约束类型，分别存储不同类型的养成数据：

---

### 1. 项目材料定义约束

#### 约束基本信息

- **ownerType**: `PROJECT`（固定）
- **ownerId**: 项目ID（关联到具体项目）
- **constraintName**: `material_definitions`（固定，表示项目材料定义）

#### params JSON 结构

params 字段是一个 JSON 对象，key 为材料唯一ID，value 为材料定义：

- **materialId**: 字符串，材料唯一ID，项目内唯一标识
- **materialName**: 字符串，材料名称（用于显示）
- **materialImageURL**: 字符串，材料图片路径
- **description**: 字符串，可选，材料描述

---

### 2. 每日掉落限制约束

#### 约束基本信息

- **ownerType**: `PROJECT`（固定）
- **ownerId**: 项目ID（关联到具体项目）
- **constraintName**: `daily_drop_limit`（固定，表示每日掉落限制）

#### params JSON 结构

params 字段是一个 JSON 对象，key 为资源ID，value 为限制配置，包含以下字段：

- **resourceId**: 字符串，资源唯一标识
- **resourceName**: 字符串，资源名称（用于显示）
- **limit**: 整数，每日掉落限制数量
- **current**: 整数，当前已获取数量

---

### 3. 每周掉落限制约束

#### 约束基本信息

- **ownerType**: `PROJECT`（固定）
- **ownerId**: 项目ID（关联到具体项目）
- **constraintName**: `weekly_drop_limit`（固定，表示每周掉落限制）

#### params JSON 结构

params 字段是一个 JSON 对象，key 为资源ID，value 为限制配置，包含以下字段：

- **resourceId**: 字符串，资源唯一标识
- **resourceName**: 字符串，资源名称（用于显示）
- **limit**: 整数，每周掉落限制数量
- **current**: 整数，当前已获取数量

---

### 4. 目标属性约束

#### 约束基本信息

- **ownerType**: `GOAL`（固定）
- **ownerId**: 目标ID（关联到具体目标）
- **constraintName**: `goal_attributes`（固定，表示目标属性扩展）

#### params JSON 结构

params 字段是一个 JSON 对象，存储目标的扩展属性，这些属性不属于核心目标表的基础字段：

- **priority**: 整数，优先级，1-5，1 最高，5 最低，用于排序和展示
- **parentId**: 整数，可为 null，父目标ID，null 表示无父目标（根目标）
- **requiredQuantity**: 整数，需要的数量，表示完成此目标需要多少个该物品/资源
- **goalType**: 字符串，目标类型，如 "equipment_craft"（装备制作）、"resource_collection"（资源获取）等

---

### 5. 背包已有资源约束

#### 约束基本信息

- **ownerType**: `PROJECT`（固定）
- **ownerId**: 项目ID（关联到具体项目）
- **constraintName**: `inventory_resources`（固定，表示背包已有资源）

#### params JSON 结构

params 字段是一个 JSON 对象，key 为资源ID，value 为已有数量：

- **resourceId**: 字符串，资源唯一标识
- **resourceName**: 字符串，资源名称（用于显示）
- **quantity**: 整数，背包中已有的数量

## 后端 API 设计

### 接口端点

所有接口统一使用前缀 `/api/v1/projects/:projectId/plugins/bqtj`

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/v1/projects/:projectId/plugins/bqtj` | 获取项目下所有养成约束数据（掉落限制、背包资源） |
| POST | `/api/v1/projects/:projectId/plugins/bqtj/constraint/:constraintName` | 更新指定约束的 params 数据 |

### 接口说明

- **GET 获取所有数据**：一次性返回项目下所有四种约束（每日掉落限制、每周掉落限制、背包已有资源）的完整数据，前端本地处理即可，不需要多次请求
- **POST 更新约束**：根据 `constraintName` 和项目信息找到对应约束记录，直接替换 params 数据，前端负责维护完整数据结构

### 请求响应格式

遵循项目统一的 RESTful 响应格式，使用标准的成功失败标记。

#### 整体布局

本插件使用 `contentPanelExtensions` 扩展点，占据三栏布局最右侧的内容区，替换原有项目导航的内容区。整体布局采用**树形缩进列表**方案（方案2），该方案实现简单快捷，复用现有目标卡片组件，用户体验接近用户习惯的markdown缩进风格。

遵循项目现有架构：
- **项目级工具入口**整合到**项目操作折叠工具栏**
- **目标级操作入口**整合到**目标折叠操作菜单**
- 选择不同项目工具后，在内容区展示对应的视图

#### 交互设计细节

##### 1. 树形缩进展示（项目概览视图）

- **缩进规则**：每一级子目标相对于父目标向右缩进 `2rem`（约32px），使用左侧边框作为视觉引导线
- **父子关系**：通过缩进和左侧引导线清晰表达层级关系，与markdown展示风格一致
- **目标卡片**：每个目标依然使用现有的目标卡片组件，从上到下排列
- **折叠展开**：支持点击卡片标题区域折叠/展开子树，方便管理大型养成树
- **进度展示**：在目标卡片内内嵌进度条，显示当前完成进度

##### 2. 工具切换机制

项目级工具入口整合到**项目操作折叠工具栏**，切换不同工具在内容区展示不同视图：

| 工具项 | 说明 | 内容区展示 |
|--------|------|------------|
| **项目概览** | 默认视图 | 树形缩进目标列表 |
| **掉落限制** | 管理每日/每周掉落限制 | 掉落限制管理面板 |
| **背包资源** | 编辑背包已有资源 | 背包资源编辑面板 |
| **进度计算** | 操作命令，重新计算进度 | 切回项目概览并刷新进度 |

##### 3. 目标创建流程

- **创建根目标**：项目概览视图底部提供"添加根目标"悬浮按钮，点击弹出创建表单
- **创建子目标**：所有目标操作（包括添加子目标）都整合到**目标折叠操作菜单**中，从菜单选择"添加子目标"
- **表单内容**：
  - 目标名称（如：氩星青蜂）
  - 目标类型（下拉选择：装备制作 / 资源收集）
  - 需要数量（数字输入）
  - 优先级（下拉选择：1-5）

##### 4. 目标卡片操作（整合到折叠菜单）

所有目标操作都整合进目标的折叠工具栏：

- **添加子目标**：在当前目标下创建缩进一级的子目标
- **编辑**：编辑目标属性（名称、数量、类型等）
- **移动**：调整顺序和层级（选择新的父目标）
- **完成**：标记目标为已完成
- **删除**：删除目标（如果有子目标需要确认）

##### 5. 掉落限制管理面板

- **视图切换**：从项目操作折叠工具栏选择"掉落限制"后内容区切换到此面板
- **分页标签**："每日掉落"和"每周掉落"两个标签页
- **列表展示**：每行一个资源，包含：
  - 资源名称
  - 当前已获取 / 限制数量
  - 下次重置时间（倒计时显示）
  - 编辑 / 删除按钮
- **添加按钮**：底部固定"添加限制"按钮，点击弹出表单

表单字段：
- 资源ID（自动生成，也可编辑）
- 资源名称
- 限制数量
- 当前已获取数量（可手动调整）

##### 6. 背包已有资源面板

- **视图切换**：从项目操作折叠工具栏选择"背包资源"后内容区切换到此面板
- **列表展示**：每行一个已有资源，包含：
  - 资源名称
  - 当前数量
  - 编辑 / 删除按钮
- **添加按钮**：底部固定"添加资源"按钮，点击弹出表单
- **表单字段**：
  - 资源ID
  - 资源名称
  - 已有数量

##### 7. 进度计算与展示

- **单个目标进度**：在目标卡片内展示进度条，计算方式：
  ```
  进度 = (已有数量 + 已掉落数量) / 需要数量
  ```
  其中：已有数量来自背包资源，已掉落数量来自掉落限制的`current`字段
- **整体进度**：单独的内容区页面
- **预计完成天数**：根据每日可获取总量，计算预计完成天数

## 后端目录结构

插件遵循项目插件目录规范，后端存放位置：

`src/plugins/bqtj/`

目录结构：
- `routes.ts` - 路由定义
- `controller.ts` - 控制器
- `service.ts` - 业务逻辑（自动重置、进度计算等）
- `types.ts` - 类型定义

因为掉落限制插件有自动重置和进度计算的业务逻辑，所以需要 Service 层处理这些复杂逻辑。
