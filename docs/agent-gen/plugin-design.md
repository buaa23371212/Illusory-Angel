# 插件设计文档

## 设计理念

核心系统保持最小化，只包含项目、目标、约束三个基础表。所有非核心功能通过插件方式扩展，插件可以在需要时逐步加入系统。

插件扩展遵循开闭原则，不修改核心表结构，通过新增数据表实现功能扩展。

## 标签系统插件

### 设计思路

标签系统用于目标的多维度分类和筛选，方便用户快速定位和过滤目标。标签支持自定义，一个目标可以绑定多个标签。

### 数据存储

需要新增两张表：标签表存储标签基本信息，关联表存储目标与标签的多对多关系。

标签包含名称和显示颜色信息，颜色用于界面可视化展示。

### 功能范围

提供目标标签绑定、多条件筛选、标签分类管理等基础功能。

### 依赖关系

依赖核心系统的目标表。

## 闹钟系统插件

### 设计思路

闹钟系统为目标提供截止日期提醒功能，支持设置单个目标的截止时间，到期前提醒用户。

### 数据存储

需要新增闹钟表，记录每个目标的截止时间、提醒提前天数、提醒开关状态等配置。

### 功能范围

提供截止日期设置、到期提醒、过期标记等功能。支持自定义提醒提前量。

### 依赖关系

依赖核心系统的目标表。

## 插件扩展规范

### 扩展原则

所有插件都通过新增数据表扩展功能，不修改核心表结构。保证核心系统稳定性。

### 外键约束

插件表与核心表保持外键关联，确保数据一致性。删除目标时级联删除插件数据。

### 独立部署

每个插件独立设计，可以按需启用或不启用，不启用不占用存储空间。
