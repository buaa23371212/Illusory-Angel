# 项目目录结构设计

## 目录设计原则

根据项目规模和开发方式选择合适的组织结构。单体仓库适合全栈独立开发，多仓库适合团队分工协作。

本项目定位为个人开发的全栈应用，采用单体仓库组织结构。

## 顶层目录划分

按职责划分为四个主要顶层目录：文档目录、后端源代码、前端源代码、脚本和工具。

文档目录存放设计文档和说明文档，后端目录存放API接口和业务逻辑，前端目录存放用户界面代码，脚本目录存放数据库初始化和部署相关脚本。

## 推荐目录结构

完整目录树结构：

```
planner/
├── docs/                    # 文档目录
│   ├── agent-gen/          # 设计文档
│   └── examples/           # 示例记录
│
├── backend/                 # 后端代码
│   ├── src/
│   │   ├── config/         # 配置文件
│   │   ├── models/         # 数据模型
│   │   ├── routes/         # 路由定义
│   │   ├── controllers/    # 控制器
│   │   ├── services/       # 业务逻辑
│   │   ├── plugins/        # 插件扩展
│   │   └── utils/          # 工具函数
│   └── tests/              # 测试代码
│
├── frontend/                # 前端代码
│   ├── src/
│   │   ├── components/     # 通用组件
│   │   ├── pages/          # 页面组件
│   │   ├── plugins/        # 前端插件
│   │   ├── utils/          # 工具函数
│   │   ├── api/            # API 调用
│   │   └── styles/         # 样式文件
│   └── public/             # 静态资源
│
└── scripts/                # 脚本工具
    └── sql/                # 数据库脚本
```

### 文档目录

存放项目相关的所有文档，包括设计文档、架构说明、示例记录等。按类型进一步分类存放。

### 后端目录

存放后端服务代码，包括配置、路由、模型、控制器、服务、工具等模块。按功能分层组织代码。

### 前端目录

存放前端用户界面代码，按现代前端框架结构组织，包含组件、页面、工具、样式等模块。

### 脚本目录

存放数据库初始化脚本、部署脚本、开发工具脚本等辅助性脚本文件。按类型分类存放。

## 单体仓库与多仓库对比

### 单体仓库优势

代码版本统一管理，依赖关系清晰，开发调试方便，便于快速迭代。适合个人开发和小型项目。

### 多仓库优势

前后端独立发布，权限控制清晰，适合大型团队协作开发。但本地开发需要同时克隆多个仓库，调试相对繁琐。

### 选择建议

个人独立开发优先选择单体仓库，开发效率更高。项目规模扩大后如果需要拆分，再拆分为多仓库也不迟。

## 插件扩展目录

后端插件按功能模块独立存放，每个插件一个目录，包含自身的数据模型和业务逻辑。前端插件也按模块独立存放，便于按需加载。

## 静态资源目录

静态资源包括图片、图标等，统一存放在前端目录中，按前端构建规范管理。
