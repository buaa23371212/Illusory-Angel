# API 技能开发文档

本文档描述了后端提供的 API 接口，用于 Agent 作为技能调用。

## 概述

后端提供了完整的 RESTful API，支持项目、目标、约束三个核心资源的增删改查操作。所有接口都遵循统一的响应格式，基础路径为 `/api/v1`。

## 技能分类

### 项目管理技能

项目是最高层级的组织单元，用于分类管理不同的计划和目标。

可用的操作包括：
- 获取项目列表，支持分页和关键词搜索
- 创建新项目
- 获取单个项目详情
- 更新项目信息
- 删除项目，级联删除下属所有目标和约束
- 获取项目下的所有约束

### 目标管理技能

目标是项目下的具体任务单元，支持层级结构和完成状态跟踪。

可用的操作包括：
- 获取指定项目下的目标列表，支持按父目标和完成状态筛选
- 在指定项目下创建新目标
- 获取单个目标详情
- 更新目标信息
- 切换目标完成状态
- 批量移动目标调整顺序
- 删除目标，级联删除下属所有约束

### 约束管理技能

约束是附加在项目或目标上的自定义条件数据，支持插件通过约束类型实现个性化功能。

可用的操作包括：
- 获取指定所有者（项目或目标）的所有约束
- 为指定所有者添加新约束
- 更新约束信息
- 删除约束

## 接口调用规范

所有请求使用 JSON 格式传输，响应也统一使用 JSON 格式。路径参数需要替换为实际值。查询参数按需添加到 URL 中。

操作成功时返回对应的数据，错误时返回错误信息和状态码。

## 基础路径

所有接口都以 `/api/v1` 作为前缀。
